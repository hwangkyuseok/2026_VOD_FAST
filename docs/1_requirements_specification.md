## 1. 프로젝트 개요

### 1.1 목적

케이블 TV 셋톱박스 방송 시스템을 웹 애플리케이션으로 에뮬레이션하는 차세대 미디어 플랫폼을 구축한다. FAST(Free Ad-Supported Streaming TV) 광고 수익 모델과 실시간 쇼핑 커머스를 핵심 수익 기반으로 한다.

### 1.2 범위

| 구분 | 포함 | 제외 |
|------|------|------|
| 플랫폼 | 웹 브라우저 전용 (데스크톱 우선) | 모바일 앱, Native TV 앱 |
| 콘텐츠 | 실시간 채널(30개), VOD(무료) | 유료 VOD 결제 기능 |
| 광고 | FAST 광고 삽입 (트랙1 VOD 한정) | 실시간 방송 광고 삽입 |
| 쇼핑 | 0번 채널 커머스 + 쇼핑 매칭 오버레이 | 직접 결제/PG 연동 |
| AI 추천 | TF-IDF + KeyBERT + 코사인 유사도 | 딥러닝 모델 자체 학습 |
| 비전 분석 | YOLOv8 (COCO 80클래스) + CLIP (컨텍스트 태그) | 자체 Fine-tuning |

### 1.3 핵심 비즈니스 규칙

> ⚠️ **절대 위반 금지 사항 — 추천·타겟팅 로직의 최우선 제약**

1. **키즈·애니메이션 콘텐츠는 추천 풀에서 절대 배제 금지.** 전체 가입자의 90% 이상이 40대 이상이더라도 키즈·애니 콘텐츠 실제 이용률이 매우 높다.
2. 키즈·애니메이션은 오히려 **주요 추천 요소**로 취급하며, 코사인 유사도 계산 시 별도 가중치(boost) 적용을 권장한다.
3. `TB_USER_PROFILE_VECTOR.kids_boost_score` 컬럼으로 키즈 추천 강제 포함 보장.
4. **원본 VOD 파일을 절대 수정하지 않는다.** 광고는 클라이언트 오버레이 방식으로만 삽입한다.

#### 2.5.1 Cold Start 처리 정책 (FR-005-CS)

| 상황 | 처리 방법 | 결과 |
|------|----------|------|
| 유저 벡터 있음 | 사전 계산된 프로필 벡터로 코사인 유사도 계산 | 개인화 추천 |
| 유저 벡터 없음 + 시청 이력 있음 | `TB_VOD_LOG` 시청 VOD 벡터 가중 평균 → 임시 프로필 생성 후 추천 | 이력 기반 추천 |
| **시청 이력 없음 (완전 신규)** | `TB_VOD_META.rate` 평점 내림차순 top 10 반환 | 인기 콘텐츠 추천 |

> `reason` 필드값: `"인기 콘텐츠 추천 (시청 이력 없음)"`, `score`: `1.0` 고정

### 2.6 FAST 광고 파이프라인 (FR-006)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-006-01 | 매주 월요일 02:00 KST 자동 배치 실행 (APScheduler, `AD_BATCH_CRON` 조정 가능) | 필수 |

| FR-006-05 | **CLIP** (`openai/clip-vit-base-patch32`)으로 20개 컨텍스트 태그 중 상위 5개 추출 (`CLIP_ENABLED` 환경변수로 비활성화 가능) | 선택 |
| FR-006-06 | YOLO + CLIP 결과 통합 → `vision_tags` (중복 제거, 최대 10개) | 필수 |
| FR-006-07 | 저움직임 구간 탐지: 씬 길이 기반 `motion_score` 계산 → 최적 삽입 포인트 최대 5개 선정 | 필수 |
| FR-006-08 | 삽입 타임스탬프: 씬 시작점 +1.0초, `insert_reason` 자동 분류 (LOW_MOTION / SCENE_BREAK / QUIET_MOMENT) | 필수 |
| FR-006-09 | 생성형 AI API로 이미지 광고 생성 (DALL-E 3, 1024×1024 PNG) | 필수 |
| FR-006-10 | 생성형 AI API로 무음 영상 광고 생성 (RunwayML/Kling, 4초 MP4) | 필수 |
| FR-006-11 | API 키 미설정 시: 이미지 → PIL 플레이스홀더, 영상 → ffmpeg 4초 검은 화면으로 graceful fallback | 필수 |
| FR-006-12 | 원본 VOD 파일 무수정 — 클라이언트 AdOverlay 오버레이 방식으로 광고 노출 | 필수 |
| FR-006-13 | 광고 오버레이 위치: 화면 하단 (`display_position=OVERLAY_BOTTOM`), 노출 시간: 4초 | 필수 |
| FR-006-14 | 동일 타임스탬프 광고는 1회만 노출 (재생 회차 내 중복 노출 방지) | 필수 |
| FR-006-15 | 파이프라인 상태를 `TB_WEEKLY_FREE_VOD.AD_PIPELINE_STATUS`에 기록 (PENDING → IN_PROGRESS → COMPLETED) | 필수 |
| FR-006-16 | 선정된 슬롯 분류를 `TB_WEEKLY_FREE_VOD.SELECTION_REASON`에 기록 (`SLOT_KIDS` / `SLOT_DOCU` / `SLOT_ENT` / `SLOT_ETC`) | 필수 |
| FR-006-17 | 슬롯 미달(선정 결과 10개 미만) 시 배치 로그에 `weekly_vod.slot_shortage` 경고 출력 | 필수 |
| FR-006-18 | 배치 실행 전 이전 주 VOD의 `IS_FREE_YN='N'` 자동 복원 (이중 과금 방지) | 필수 |

## 5. 제약사항 및 가정

### 5.1 기술 제약

- PostgreSQL DB는 외부 서버(`YOUR_SERVER_IP:5432/hv02`)에 이미 운영 중 — 스키마 변경 시 사전 확인 필요
- VOD 원본 파일 경로: 컨테이너 내부 `/app/data/vod` (읽기 전용 마운트), 없을 경우 더미 씬으로 계속 진행
- 생성형 AI API Key는 외부 서비스 의존 — 미설정 시 플레이스홀더로 fallback (배치 중단 없음)
- CLIP 모델은 약 600MB, KR-ELECTRA는 약 430MB — 최초 기동 시 인터넷 연결 필요
- TF-IDF 모델은 `POST /admin/vod_proc` 최초 1회 호출 후 tfidf.pkl이 생성되어야 추천이 정상 작동
- 브라우저 환경에서 HLS 스트리밍 지원 필수 (`hls.js` 라이브러리 사용)

### 5.2 가정

- 사용자는 데스크톱 환경에서 키보드 조작을 기반으로 TV 에뮬레이터를 사용한다고 가정
- `TB_PROD_INFO` 기준 상품 데이터는 외부 크롤링 배치를 통해 사전 적재되어 있다고 가정
- VOD 스트리밍 URL은 `TB_VOD_META.MOBILE_WATCH_URL` 또는 별도 CDN에 존재한다고 가정
- `TB_VOD_META.is_free_yn='Y'`인 VOD만 무료 서비스 대상으로 가정

| 편성표 | 1~30번 채널 목록 오버레이 (L 키 단축키) |
| kids_boost_score | 유저별 키즈·애니 장르 추천 가중치 (0.0~1.0) |
| vision_tags | YOLO 객체 감지 + CLIP 컨텍스트 태그를 통합한 광고 키워드 목록 |
| Cold Start | 시청 이력이 전혀 없는 신규 유저에게 평점 기반 인기 콘텐츠를 추천하는 전략 |
| motion_score | 씬 길이 기반 움직임 지표 (낮을수록 정적 = 광고 삽입 적합) |
| 슬라이딩 윈도우 | 전체 VOD 목록 중 일부만 화면에 표시하고 포커스 이동에 따라 뷰 범위를 조정하는 방식 |
| tfidf.pkl | TF-IDF 벡터라이저 모델을 pickle 직렬화하여 Docker 볼륨에 영속화한 파일 |

| seasonal_themes | 12개월 시즌별 SMRY/TTL 키워드 딕셔너리. `ad-batch/app/seasonal_themes.py`에서 관리 |
| DISP_RTM | `TB_VOD_META` 컬럼. 영상 상영 시간 문자열 (`HH:MM:SS` 형식, VARCHAR(20)) |
