## 1. 프로젝트 개요

### 1.1 목적

케이블 TV 셋톱박스 방송 시스템을 웹 애플리케이션으로 에뮬레이션하는 차세대 미디어 플랫폼을 구축한다. FAST(Free Ad-Supported Streaming TV) 광고 수익 모델과 실시간 쇼핑 커머스를 핵심 수익 기반으로 한다.

### 1.2 범위

| 구분 | 포함 | 제외 |
|------|------|------|
| 플랫폼 | 웹 브라우저 전용 (데스크톱 우선) | 모바일 앱, Native TV 앱 |
| 콘텐츠 | 실시간 채널(30개), VOD(무료) | 유료 VOD 결제 기능 |
| 광고 | FAST 광고 삽입 (트랙1 VOD 한정) | 실시간 방송 광고 삽입 |
| 쇼핑 | 0번 채널 커머스 + 쇼핑 매칭 오버레이 | 직접 결제/PG 연동 |

### 1.3 핵심 비즈니스 규칙

> ⚠️ **절대 위반 금지 사항 — 추천·타겟팅 로직의 최우선 제약**

1. **키즈·애니메이션 콘텐츠는 추천 풀에서 절대 배제 금지.** 전체 가입자의 90% 이상이 40대 이상이더라도 키즈·애니 콘텐츠 실제 이용률이 매우 높다.
2. 키즈·애니메이션은 오히려 **주요 추천 요소**로 취급하며, 코사인 유사도 계산 시 별도 가중치(boost) 적용을 권장한다.
3. `TB_USER_PROFILE_VECTOR.kids_boost_score` 컬럼으로 키즈 추천 강제 포함 보장.

## 2. 기능 요구사항

### 2.1 사용자 초기 설정 (FR-001)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|

| FR-001-03 | `tv_user_id` 쿠키/스토리지 미존재 시 항상 `/setup`으로 강제 이동 | 필수 |

### 2.2 실시간 채널 시청 (FR-002)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-002-01 | `/channel` 진입 시 기본 채널은 **0번 채널(HV 쇼핑TV)** | 필수 |
| FR-002-02 | 채널 번호는 0번(커머스) ~ 30번(가상 채널) 구성 | 필수 |
| FR-002-03 | 채널 전환(Zapping) 목표 응답시간: **500ms 이내** | 필수 |
| FR-002-04 | 키보드 방향키 ▲/▼로 채널 전환 | 필수 |
| FR-002-05 | 1~30번 채널 화면에서 하단 컨트롤 바 미표시 | 필수 |
| FR-002-06 | 1~30번 채널에서 `L` 키로 채널 편성표 오버레이 표시/숨김 | 필수 |
| FR-002-07 | 채널 편성표 오버레이에서 `ESC` 키로 닫기 | 필수 |
| FR-002-08 | 시청 세션 시작/종료 이력을 `TB_WATCH_SESSION`에 기록 | 필수 |

|----|---------|---------|

| FR-003-07 | 상품 Enter 선택 시: 가격 200,000원 미만 → 구매 모달, 이상 → 상담 모달 | 필수 |

## 4. 사용자 스토리

### 4.1 일반 시청자

```
시나리오 1 — 첫 방문
AS A  첫 방문 시청자
I WANT TO  사용자 ID를 설정하고 TV를 시청하고 싶다
SO THAT  개인화된 VOD 추천을 받을 수 있다

수락 기준:
  - /channel 접근 시 tv_user_id 미설정이면 /setup으로 리다이렉트
  - 설정 완료 후 /channel?ch=0(HV 쇼핑TV)로 자동 이동

시나리오 2 — 채널 시청
AS A  TV 시청자
I WANT TO  채널을 자유롭게 전환하고 싶다
SO THAT  다양한 콘텐츠를 빠르게 탐색할 수 있다

수락 기준:
  - ▲/▼ 키 입력 시 500ms 이내 채널 전환
  - L 키로 편성표 오버레이 즉시 표시
  - 0번 채널에서는 커머스 전용 화면 표시

시나리오 3 — 쇼핑 채널
AS A  쇼핑에 관심 있는 시청자
I WANT TO  0번 채널에서 렌탈 상품을 탐색하고 싶다
SO THAT  마음에 드는 상품을 바로 상담/구매할 수 있다

수락 기준:
  - 상품 카드에 이미지, 상품명, 가격(렌탈료/월) 표시
  - Enter 키로 상품 선택 → 가격에 따라 구매/상담 모달 표시
  - VOD 메뉴 선택 시 /vod로 이동
```

### 4.2 관리자

```
시나리오 4 — VOD 추천 관리
AS A  플랫폼 운영자
I WANT TO  NLP 추천 모델을 주기적으로 업데이트하고 싶다
SO THAT  고객에게 항상 최신 개인화 추천을 제공할 수 있다

수락 기준:
  - /admin/vod_proc API 호출로 전체 VOD 벡터 재계산 가능
  - 특정 유저 프로필 벡터 수동 재계산 지원
  - 키즈·애니메이션 장르 가중치 환경변수로 조절 가능(KIDS_BOOST_SCORE)
```

## 5. 제약사항 및 가정

### 5.1 기술 제약

- PostgreSQL DB는 외부 서버(`YOUR_SERVER_IP:5432/hv02`)에 이미 운영 중 — 스키마 변경 시 사전 확인 필요
- VOD 원본 파일 경로: 컨테이너 내부 `/app/data/vod` (읽기 전용 마운트)
- 생성형 AI API Key는 외부 서비스 의존 — API 장애 시 광고 생성 배치 실패 가능
- 브라우저 환경에서 HLS 스트리밍 지원 필수 (`hls.js` 라이브러리 사용)

### 5.2 가정

- 사용자는 데스크톱 환경에서 키보드 조작을 기반으로 TV 에뮬레이터를 사용한다고 가정
- `TB_PROD_INFO` 기준 상품 데이터는 외부 크롤링 배치를 통해 사전 적재되어 있다고 가정
- VOD 스트리밍 URL은 `TB_VOD_META.MOBILE_WATCH_URL` 또는 별도 CDN에 존재한다고 가정

## 6. 용어 정의

| 용어 | 정의 |
|------|------|
| FAST | Free Ad-Supported Streaming TV. 광고 기반 무료 스트리밍 모델 |
| Zapping | 채널을 빠르게 전환하는 행위 |
| 트랙 1 | 금주의 무료 VOD. 전 고객 공통 제공, FAST 광고 삽입 |
| 트랙 2 | 당신을 위한 무료 VOD. 개인화 추천, 광고 없음 |
| 오버레이 | 원본 영상 위에 UI 레이어를 겹쳐 표시하는 방식 |
| 렌탈 상품 | `TB_PROD_INFO.is_rental = 'Y'`인 월 렌탈료 기반 상품 |
| 커머스 채널 | 0번 채널. HV 쇼핑TV. `TB_PROD_INFO` 상품 목록 표시 |
| 편성표 | 1~30번 채널 목록 오버레이 (L 키 단축키) |
| kids_boost_score | 유저별 키즈·애니 장르 추천 가중치 (0.0~1.0) |
