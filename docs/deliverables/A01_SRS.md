## 1. 프로젝트 개요

### 1.1 목적

케이블 TV 셋톱박스 방송 시스템을 **웹 애플리케이션으로 에뮬레이션**하는 차세대 미디어 플랫폼을 구축한다.  
FAST(Free Ad-Supported Streaming TV) 광고 수익 모델과 실시간 쇼핑 커머스를 핵심 수익 기반으로 한다.

### 1.2 프로젝트 범위

| 구분 | 포함 | 제외 |
|------|------|------|
| 플랫폼 | 웹 브라우저 전용 (데스크톱 우선) | 모바일 앱, Native TV 앱 |
| 콘텐츠 | 실시간 채널(30개), VOD(무료) | 유료 VOD 결제 기능 |
| 광고 | FAST 광고 삽입 (트랙1 VOD 한정) | 실시간 방송 광고 삽입 |
| 쇼핑 | 0번 채널 커머스 + 쇼핑 매칭 오버레이 | 직접 결제/PG 연동 |
| AI 추천 | TF-IDF + KeyBERT + 코사인 유사도 | 딥러닝 모델 자체 학습 |
| 비전 분석 | YOLOv8 + CLIP | 자체 Fine-tuning |

### 1.3 이해관계자

| 이해관계자 | 역할 | 기대 |
|-----------|------|------|
| 일반 시청자 | 채널 시청, VOD 탐색 및 재생 | 개인화된 콘텐츠 경험 |
| 쇼핑 이용자 | 0번 커머스 채널 상품 탐색 | 간편한 상담/구매 연결 |
| 플랫폼 운영자 | NLP 모델 관리, 광고 파이프라인 모니터링 | 자동화된 수익 모델 운영 |

## 2. 핵심 비즈니스 규칙

> ⚠️ **절대 위반 금지 — 추천·타겟팅 로직의 최우선 제약**

1. **키즈·애니메이션 콘텐츠는 추천 풀에서 절대 배제 금지.** 오히려 주요 추천 요소로 취급.
2. 키즈·애니메이션은 코사인 유사도 계산 시 별도 `kids_boost_score` 가중치 적용.
3. **원본 VOD 파일을 절대 수정하지 않는다.** 광고는 클라이언트 오버레이 방식으로만 삽입.

## 3. 기능 요구사항

### 3.1 사용자 초기 설정 (FR-001)

| ID | 요구사항 | 우선순위 |
|----|---------|---------| 
| FR-001-01 | 최초 진입 시 `/setup` 페이지로 리다이렉트 | 필수 |
| FR-001-02 | 사용자 ID 입력 후 localStorage(`tv_user_id`)에 저장 | 필수 |
| FR-001-03 | `tv_user_id` 미존재 시 항상 `/setup`으로 강제 이동 | 필수 |
| FR-001-04 | 설정 완료 후 `/channel`(0번 채널)로 자동 이동 | 필수 |

### 3.2 실시간 채널 시청 (FR-002)

| ID | 요구사항 | 우선순위 |
|----|---------|---------| 
| FR-002-01 | `/channel` 진입 시 기본 채널은 **0번 채널(HV 쇼핑TV)** | 필수 |
| FR-002-02 | 채널 번호는 0번(커머스) ~ 30번(가상 채널) 구성 | 필수 |
| FR-002-03 | 채널 전환(Zapping) 목표 응답시간: **500ms 이내** | 필수 |
| FR-002-04 | 키보드 방향키 ▲/▼로 채널 전환 | 필수 |
| FR-002-05 | 1~30번 채널 화면에서 하단 컨트롤 바 미표시 | 필수 |
| FR-002-06 | `L` 키로 채널 편성표 오버레이 표시/숨김 | 필수 |
| FR-002-07 | 시청 세션 이력을 `TB_WATCH_SESSION`에 기록 | 필수 |
| FR-002-08 | 채널 스트림 URL은 Redis 캐시 우선 사용 | 필수 |

### 3.3 0번 채널 TV 커머스 (FR-003)

| ID | 요구사항 | 우선순위 |
|----|---------|---------| 
| FR-003-01 | 0번 채널 진입 시 커머스 화면(CommerceChannel) 표시 | 필수 |
| FR-003-02 | `TB_PROD_INFO`에서 상품 목록 실시간 조회 (최대 20개) | 필수 |
| FR-003-03 | 가격 우선순위: `sale_price` → (NULL이면) `monthly_rental_fee` | 필수 |
| FR-003-04 | 렌탈 상품(`is_rental=Y`)의 가격은 "N원/월" 형식으로 표시 | 필수 |
| FR-003-05 | `B` 키로 좌측 사이드바 메뉴 표시/숨김 | 필수 |
| FR-003-06 | 상품 목록은 `srl_no` 순서 기준으로 정렬 | 필수 |
| FR-003-07 | 상품 Enter 선택 시: 가격 200,000원 미만 → 구매 모달, 이상 → 상담 모달 | 필수 |

### 3.4 VOD 페이지 구성 (FR-004)

| ID | 요구사항 | 우선순위 |
|----|---------|---------| 
| FR-004-01 | VOD 페이지는 **광고 배너 / 금주의 무료 VOD / 추천 VOD** 3단 레이아웃으로 구성 | 필수 |
| FR-004-02 | 모든 섹션이 한 화면에 표시, 수직 스크롤 비활성화 | 필수 |
| FR-004-03 | 최초 포커스는 **금주의 무료 VOD 섹션 첫 번째 항목**에 위치 | 필수 |

#### 3.4.1 금주의 무료 VOD — 트랙1 (FR-004-T1)

| ID | 요구사항 | 우선순위 |
|----|---------|---------| 
| FR-004-T1-01 | 전 고객 공통, 주 1회(매주 월요일) 자동 갱신 | 필수 |
| FR-004-T1-02 | 총 **10개** VOD 보유, 화면에는 **6개씩 슬라이딩 윈도우** 표시 | 필수 |
| FR-004-T1-03 | VOD 선택 시 **전체화면 플레이어**로 전환, FAST 광고 오버레이 적용 | 필수 |

**v2 큐레이션 전략** (ad-batch 배치, 매주 월요일 02:00):

| 항목 | 내용 |
|------|------|
| 하드 필터 | `USE_FL=1`, `SVC_APY_FL=1`, `THMBNL_FL=1`, `IS_FREE_YN='N'`, `DISP_RTM >= '00:20:00'` |
| 슬롯 배분 | 키즈/애니 ≤3개 + 교양/다큐 ≤4개 + 예능/음악 ≤2개 + 기타 ≤1개 |
| 소프트 점수 | 인기작 +15, 고화질 +10, 키즈 1화 +15, 4060 키워드 +20, 월별 테마 +30 |
| 중복 제거 | `SUPER_ASSET_NM` 기준 시리즈당 최고점 1편 선정 |
| IS_FREE_YN 복원 | 신규 배치 전 이전 주 VOD `IS_FREE_YN='N'` 자동 복원 |

#### 3.4.2 추천 VOD — 트랙2 (FR-004-T2)

| ID | 요구사항 | 우선순위 |
|----|---------|---------| 
| FR-004-T2-01 | 고객 개인화 추천, 실시간 조회 | 필수 |
| FR-004-T2-02 | 총 **10개** VOD, 화면에는 **6개씩 슬라이딩 윈도우** 표시 | 필수 |
| FR-004-T2-03 | 키즈·애니메이션 장르는 연령과 무관하게 추천 풀에 항상 포함 | 필수 |

### 3.5 NLP 기반 개인화 추천 엔진 (FR-005)

| ID | 요구사항 | 우선순위 |
|----|---------|---------| 
| FR-005-01 | TF-IDF + KeyBERT로 VOD 메타데이터 텍스트 벡터화 | 필수 |
| FR-005-02 | 벡터화 소스: `TTL + GENRE + DESCRIPTION + HASH_TAG + SMRY` | 필수 |
| FR-005-03 | TF-IDF 모델을 pickle 파일(`tfidf.pkl`)로 영속화 — 재시작 시 자동 로드 | 필수 |
| FR-005-04 | 고객 시청이력(`TB_VOD_LOG`) 기반 유저 프로필 벡터 생성 | 필수 |
| FR-005-05 | 코사인 유사도 기반 상위 10개 VOD 선정 | 필수 |
| FR-005-06 | 키즈·애니메이션 장르에 `kids_boost_score` 가중치 추가 | 필수 |
| FR-005-07 | 한국어 사전학습 모델: `snunlp/KR-ELECTRA-discriminator` | 필수 |

**Cold Start 처리 정책 (FR-005-CS)**:

| 상황 | 처리 방법 | 결과 |
|------|----------|------|
| 유저 벡터 있음 | 사전 계산된 프로필 벡터로 코사인 유사도 계산 | 개인화 추천 |
| 유저 벡터 없음 + 시청 이력 있음 | `TB_VOD_LOG` 기반 임시 프로필 생성 | 이력 기반 추천 |
| 시청 이력 없음 (완전 신규) | `RATE` 평점 내림차순 top 10 반환 | 인기 콘텐츠 추천 |

### 3.6 FAST 광고 파이프라인 (FR-006)

| ID | 요구사항 | 우선순위 |
|----|---------|---------| 
| FR-006-01 | 매주 월요일 02:00 KST 자동 배치 실행 | 필수 |
| FR-006-02 | PySceneDetect ContentDetector로 씬 분할 (비디오 프레임만, threshold=30.0) | 필수 |
| FR-006-03 | YOLOv8n (COCO 80클래스, conf≥0.5)으로 키프레임 내 객체 감지 | 필수 |
| FR-006-04 | CLIP(`openai/clip-vit-base-patch32`)으로 20개 컨텍스트 태그 중 상위 5개 추출 | 선택 |
| FR-006-05 | YOLO + CLIP 결과 통합 → `vision_tags` (중복 제거, 최대 10개) | 필수 |
| FR-006-06 | 생성형 AI API로 이미지 광고 생성 (DALL-E 3, 1024×1024 PNG) | 필수 |
| FR-006-07 | 생성형 AI API로 무음 영상 광고 생성 (RunwayML/Kling, 4초 MP4) | 필수 |
| FR-006-08 | API 키 미설정 시: 이미지 → PIL 플레이스홀더, 영상 → ffmpeg 검은 화면 fallback | 필수 |
| FR-006-09 | 원본 VOD 파일 무수정 — 클라이언트 AdOverlay 오버레이 방식으로 광고 노출 | 필수 |
| FR-006-10 | 광고 오버레이 위치: 화면 하단, 4초 표시, 동일 타임스탬프 1회만 노출 | 필수 |
| FR-006-11 | SELECTION_REASON 슬롯별 기록: `SLOT_KIDS`/`SLOT_DOCU`/`SLOT_ENT`/`SLOT_ETC` | 필수 |
| FR-006-12 | 슬롯 미달(10개 미만) 시 `weekly_vod.slot_shortage` 경고 로그 출력 | 필수 |
| FR-006-13 | 배치 실행 전 이전 주 VOD `IS_FREE_YN='N'` 자동 복원 | 필수 |

### 3.7 실시간 쇼핑 매칭 오버레이 (FR-007)

## 4. 비기능 요구사항

### 4.1 성능 (NFR-P)

| ID | 항목 | 목표값 |
|----|------|--------|
| NFR-P-01 | 채널 전환(Zapping) 응답시간 | < 500ms |
| NFR-P-02 | VOD 개인화 추천 API 응답시간 | < 200ms |
| NFR-P-03 | Cold Start 추천 응답시간 | < 100ms |
| NFR-P-04 | FAST 광고 오버레이 렌더링 | < 100ms |
| NFR-P-05 | 동시 접속자 | 100명 이상 |

### 4.2 보안 (NFR-S)

| ID | 요구사항 |
|----|---------|
| NFR-S-01 | 모든 접속 정보는 `.env` 파일에서만 관리 — 코드 하드코딩 금지 |
| NFR-S-02 | CORS 오리진은 `CORS_ORIGINS` 환경변수로 제한 |
| NFR-S-03 | 생성형 AI API 키는 ad-batch 서비스에서만 사용 |
| NFR-S-04 | 고객 식별자는 `SHA2_HASH` 사용으로 원본 개인정보 비노출 |

### 4.3 가용성 (NFR-A)

| ID | 요구사항 |
|----|---------|
| NFR-A-01 | backend-api 헬스체크 엔드포인트: `GET /health` |
| NFR-A-02 | nlp-api 헬스체크에 `tfidf_ready` 상태 포함 |
| NFR-A-03 | API 키 미설정 시 배치 파이프라인은 플레이스홀더로 계속 진행 |
| NFR-A-04 | CLIP 로드 실패 시 YOLO만으로 graceful fallback |

### 4.4 유지보수성 (NFR-M)

| ID | 요구사항 |
|----|---------|
| NFR-M-01 | 서비스별 독립된 Dockerfile로 개별 배포 가능 |
| NFR-M-02 | 구조화된 JSON 로깅 (structlog, 전 서비스 통일) |
| NFR-M-03 | 배치 스케줄은 `AD_BATCH_CRON` 환경변수로 코드 변경 없이 조정 가능 |
| NFR-M-04 | 월별 테마 키워드는 `seasonal_themes.py` 수정만으로 변경 가능 |

## 5. 제약사항 및 가정

| 구분 | 내용 |
|------|------|
| DB | PostgreSQL 외부 운영 서버(`YOUR_SERVER_IP:5432/hv02`) — 스키마 변경 시 사전 확인 필요 |
| VOD 파일 | 컨테이너 내부 `/app/data/vod` (읽기 전용 마운트), 없을 경우 더미 씬으로 진행 |
| AI API | 생성형 AI API Key는 외부 서비스 의존 — 미설정 시 플레이스홀더로 fallback |
| 모델 크기 | CLIP ~600MB, KR-ELECTRA ~430MB — 최초 기동 시 인터넷 연결 필요 |
| 브라우저 | HLS 스트리밍 지원 필수 (`hls.js` 라이브러리 사용) |
